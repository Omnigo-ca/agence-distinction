# Google Doc de révision client — structure

Doc envoyé avec la V1 (étape 9). Objectif : récolter des commentaires structurés, section par section, en une seule ronde. Utiliser les **onglets (tabs)** de Google Docs : 1 onglet par section du site.

## Nommage et emplacement

- Nom : `{Client} — Révision site web V1`
- Emplacement : dossier Drive du client (créé à l'étape 5)
- Partage : commentateur pour le client

## Onglet 1 — « Lisez-moi »

```
Bonjour {Prénom},

Voici la première version de votre site web : {LIEN PREVIEW}

Comment réviser :
1. Ouvrez le site et parcourez-le (sur ordinateur ET sur votre cellulaire).
2. Chaque onglet de ce document correspond à une section de votre site.
3. Écrivez vos commentaires directement dans l'onglet concerné
   (textes à changer, photos à remplacer, couleurs, etc.).
4. Soyez précis : « Remplacer "X" par "Y" » est plus efficace que « je n'aime pas ça ».

Votre forfait inclut une (1) ronde de révisions. Regroupez tous vos
commentaires ici — nous appliquerons tout d'un coup pour la version finale.

Merci de nous retourner vos commentaires d'ici le {DATE — 5 jours ouvrables}.

Éléments à confirmer de votre côté :
- {liste des [À CONFIRMER] de la V1}
```

## Onglets suivants — 1 par section du site

Créer un onglet pour chaque section réellement présente sur le site, dans le même ordre. Structure de chaque onglet :

```
SECTION : {Nom — ex. Accueil (haut de page)}

Ce qu'on y trouve actuellement :
- Titre : « {headline actuelle} »
- Texte : « {texte actuel} »
- Bouton : « {CTA} »
- Image : {description}

VOS COMMENTAIRES :
(écrivez ici)
```

Onglets types : Accueil (hero) · Nos services · Promotions · À propos · Réalisations · Témoignages · FAQ · Contact · Pied de page · Commentaires généraux (couleurs, style, autre).

## Au retour des commentaires (étape 10)

1. Copier tous les commentaires dans une liste de tâches (Asana ou directement dans Cursor).
2. Traiter 100 % des points — si un point est refusé (hors scope 1000 $, mauvaise pratique), l'expliquer dans le courriel V2.
3. Demandes hors scope (nouvelle page, boutique, rédaction de blogue...) → transférer au vendeur (opportunité d'upsell).
